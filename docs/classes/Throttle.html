Class Throttle

The Throttle rate-limits access to a resource over a moving time window using a 'leaky bucket' algorithm.

waitTime

  • get waitTime(): number
    • Defined in src/throttle.ts:65

    Returns the time (ms) until a next request can be honored (plus 1ms if there's a wait). This is useful in case multiple throttles need to be checked before a request can be consumed. Contrary to nextSlot, this getter updates the throttle's state before it produces a value.

    Returns number

Methods

consume

  • consume(): void
    • Defined in src/throttle.ts:91

    Consumes one unit of capacity. Should only be called if {@link Throttle#waitTime waitTime} > 0.

    example
    function doSomething() {
        const delay = myThrottle.waitTime;
        if (delay > 0)
            return "I can't do this right now, but in " + delay + "ms I can.";

        myThrottle.consume();
        // Do it.
        return "I did it.";
    }

    Returns void

permit

  • permit(): Promise<any>
    • Defined in src/throttle.ts:110

    Returns a promise that is guaranteed to resolve (in FIFO order), but not sooner than the throttle allows. For certain use cases this provides a more convenient alternative compared to using the Throttle.waitTime and Throttle.consume pair.

    example
    async function doSomething() {
        await myThrottle.permit();   // Resolves immediately or as soon as possible.
        // Do it.
        return "I did it.";
    }

    Returns Promise<any>
